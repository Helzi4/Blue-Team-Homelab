# Detection Analysis

## Detection Sources

The activity was detected using multiple telemetry sources:

Sysmon  
Wazuh  
LimaCharlie

## Key Detection Events

### Sysmon Event ID 11 — File Creation

File created:

C:\Users\Admin\AppData\Roaming\taskhostw.exe

Creator process:

C:\Windows\System32\cmd.exe

This indicates an executable binary was written to a user writable directory.

---

### Sysmon Event ID 1 — Process Creation

Process executed:

C:\Users\Admin\AppData\Roaming\taskhostw.exe

Parent process:

C:\Windows\System32\cmd.exe

Command line captured:

cmd.exe /c copy %windir%\System32\WindowsPowerShell\v1.0\powershell.exe %APPDATA%\taskhostw.exe /Y

This confirms the executable was created by copying PowerShell and renaming it.

---

### Temporary Script Creation

During execution, the process created a temporary PowerShell script:

C:\Users\Admin\AppData\Local\Temp\__PSScriptPolicyTest_*.ps1

These files are automatically generated by PowerShell to verify execution policy and are typically deleted immediately after execution.

This artifact confirms the masqueraded executable was executing PowerShell functionality.

---

### Wazuh Alerts

Wazuh generated alerts indicating:

- executable file dropped in suspicious directory
- abnormal command shell activity
- suspicious command execution

These alerts helped identify the initial suspicious behavior.

---

## Detection Opportunities

This behavior can be detected by monitoring:

- Windows system process names executed outside System32
- Executables created in user writable directories
- Correlation between file creation and immediate execution
- Temporary PowerShell script generation in user Temp directories
