# Case 03 — Masquerading: Renamed PowerShell Executed as taskhostw.exe

## Scenario

During routine monitoring, a suspicious executable named `taskhostw.exe` was observed running from a user directory rather than the standard Windows system directory.

The legitimate Windows binary normally runs from:

C:\Windows\System32\taskhostw.exe

However, telemetry revealed execution from:

C:\Users\Admin\AppData\Roaming\taskhostw.exe

Investigation revealed that the binary was created by copying the legitimate Windows PowerShell executable and renaming it to mimic a trusted Windows process.

Command observed:

cmd.exe /c copy %windir%\System32\WindowsPowerShell\v1.0\powershell.exe %APPDATA%\taskhostw.exe /Y

The binary was then executed from the user AppData directory.

During execution, the process created a temporary PowerShell script inside the user Temp directory:

C:\Users\Admin\AppData\Local\Temp\__PSScriptPolicyTest_*.ps1

This file is typically generated by PowerShell during execution policy validation.

The behavior simulates **MITRE ATT&CK Technique T1036.003 — Masquerading: Rename Legitimate Utilities**, where attackers disguise tools using trusted process names to evade detection.
